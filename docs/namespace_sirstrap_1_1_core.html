class Sirstrap.Core.PackageManager ( HttpClient httpClient)

Coordinates the download and assembly of Roblox application packages, handling manifest processing, package downloads, and final ZIP archive creation.

Downloads a macOS binary as a single ZIP archive file.

Parameters
downloadConfigurationConfiguration specifying the binary type, version, and other download parameters.
Returns
A task representing the asynchronous download operation.

The binary type determines which file is downloaded:

  • "MacPlayer" downloads RobloxPlayer.zip
  • Other types download RobloxStudioApp.zip The file is saved directly to the configured output location.

Downloads the package manifest file for the specified version.

Parameters
downloadConfigurationConfiguration specifying the version, channel, and other download parameters.
Returns
A task representing the asynchronous operation. The task result contains the manifest content as a string.

Processes the manifest content to download and assemble all required packages.

Parameters
manifestContentThe raw manifest content as a string.
downloadConfigurationConfiguration specifying the version, channel, and other download parameters.
Returns
A task representing the asynchronous processing operation.

This method:

  1. Parses the manifest content
  2. Validates that the manifest format is correct
  3. Initiates the download and assembly of all packages listed in the manifest

The operation will terminate early if the manifest is invalid.

Assembles the final ZIP archive by processing all packages from the manifest.

Parameters
manifestThe parsed manifest containing the list of packages to process.
downloadConfigurationConfiguration specifying the version and output file parameters.
Returns
A task representing the asynchronous assembly operation.

Creates a new ZIP archive at the specified output location, adds default settings, then downloads and processes all packages specified in the manifest.

Adds default application settings to the ZIP archive.

Parameters

Creates an AppSettings.xml file with base configuration for content folder and base URL settings.

Initiates the parallel download and processing of all packages from the manifest.

class Sirstrap.Core.VersionManager ( HttpClient httpClient)

Manages version information for Roblox deployments, providing functionality to retrieve the latest versions from both SirHurt and official Roblox APIs, compare them, and help users select the appropriate version.

Retrieves the latest version for the specified binary type by checking both SirHurt and official Roblox APIs, and prompting for user selection if versions differ.

Parameters
binaryTypeThe type of binary to get the version for (e.g., "WindowsPlayer").
Returns
A task representing the asynchronous operation. The task result contains the selected version string, or an empty string if version retrieval fails.

This method:

  1. Attempts to retrieve version information from both SirHurt and Roblox APIs
  2. If both versions match, returns that version automatically
  3. If versions differ, prompts the user to choose which version to use
  4. Returns an empty string if version retrieval fails from either source

Retrieves the latest Roblox version from the SirHurt API for the specified binary type.

Parameters
binaryTypeThe type of binary to get the version for (e.g., "WindowsPlayer").
Returns
A task representing the asynchronous operation. The task result contains the version string from the SirHurt API, or an empty string if retrieval fails.

Currently, only "WindowsPlayer" binary type is supported for SirHurt version retrieval. The method parses the JSON response to extract the "roblox_version" field.

Retrieves the latest Roblox version from the official Roblox API.

Returns
A task representing the asynchronous operation. The task result contains the version string from the Roblox API, or an empty string if retrieval fails.

The method queries the Roblox client settings CDN and parses the JSON response to extract the "clientVersionUpload" field, which contains the latest version.

Ensures a version string is in the standardized "version-X.Y.Z.W" format.

Parameters
versionThe version string to normalize.
Returns
A normalized version string that always starts with "version-".

If the input version already starts with "version-", it is returned unchanged. Otherwise, "version-" is prepended to the input string.

Determines the appropriate SirHurt API URL for the specified binary type.

Parameters
binaryTypeThe type of binary to get the version for.
Returns
The URL to the SirHurt API for the specified binary type, or an empty string if the binary type is not supported.

Currently, only "WindowsPlayer" binary type is supported, which returns the SirHurt V5 status API URL.
